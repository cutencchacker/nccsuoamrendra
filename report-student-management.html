Student Management System

Project report: records, attendance, grades, and workflows

Overview

A CRUD-centric academic system to manage students, courses, attendance, grades, and reporting.

Objectives

  • Centralize student profiles and course enrollment.
  • Track attendance and grades consistently.
  • Generate quick reports for admins/teachers.

Architecture & Stack

Java Spring / JDBC MySQL REST APIs HTML/CSS/JS (basic UI)

Layered architecture: controller → service → DAO → DB; role-based actions for admin/teacher.

Features

Student CRUD & course enrollment
Attendance marking and summaries
Grades entry, GPA calculation
Reports (per-student, per-course)
Role-based access (admin/teacher)

Implementation Notes

  • Relational schema: students, courses, enrollments, attendance, grades.
  • Transactions for grade/attendance bulk updates.
  • Validation on roll numbers, course codes.
  • Pagination for large student lists.

Testing

  • Unit tests on service layer (enrollment, GPA calc).
  • Integration tests on CRUD endpoints.
  • Manual QA for role-based flows.

Future Work

  • Attendance via QR / biometric.
  • Exports (PDF/CSV) and email reports.
  • LMS integration and notifications.
