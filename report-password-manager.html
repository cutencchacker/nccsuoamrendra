Password Manager

Project report: encryption, vault flows, and security notes

Overview

Desktop password manager with encrypted vault, password generator, and secure storage guarded by a master password.

Objectives

  • Strong encryption for stored credentials.
  • Usable vault search and copy-to-clipboard.
  • Password generator with strength checks.

Crypto & Stack

Python PyCryptodome / Fernet (AES) SQLite Tkinter UI Hashing (PBKDF2)

Master password derives key; vault entries encrypted at rest; salts + iterations to slow brute force.

Features

Encrypted vault (AES via Fernet)
Password generator + strength meter
Search and copy credentials
Export/Import (encrypted backup)

Implementation Notes

  • PBKDF2 for key derivation from master password.
  • Per-entry IV; ciphertext + metadata stored in SQLite.
  • Clipboard auto-clear after short timeout.
  • Optional password strength scoring before save.

Security Considerations

  • No master-password recovery; user responsibility.
  • Encourage long passphrases; rate-limit unlock attempts.
  • Disk encryption recommended for host machine.

Future Work

  • Browser extension for auto-fill.
  • 2FA/TOTP vault support.
  • Cloud sync with end-to-end encryption.
